Classe CGame

Classe correspondant à l'interface graphique du jeu, elle utilise tous les elements (zone de jeu, personnage, arrivée, obstacles, etc...). C'est le cœur du jeu : avec utilisation de Thread pour la gestion des parties et la gestion des sauvegardes.

Module : from CGame import CGame

Classe(s) fille(s) : /

Documentation des membres

CGame : fenetre

Correspond à la fenêtre principale, elle contient tous les menus et autres elements du jeu

CGame : map

Correspond à l'espace de jeu, elle contient les objets CPersonnage, CArrivee et CObstacle

CGame : personnage

Correspond au joueur, objet de forme cubique pouvant se deplacer dans l'objet CMap et intéragir avec ses autes éléments (CArrivee et CObstacle)

CGame : arrivee

Correspond à l'arrivée, objet de forme cubique sur lequel le joueur CPersonnage doit se placer

CGame : obstacles[]

Représente un tableau de CObstacle, il contient tous les obstacles pour permettre une gestion plus facile et rapide des intersections avec un éventuel CPersonnage

CGame : game_status_canvas

Cadre de type tkinter.Canvas correspondant au menu de la gestion du jeu, il contient les différents boutons permettant de contrôler le jeu (startBTN, restartBTN, nextBTN, pauseBTN, saveBTN, show_menuBTN)

CGame : startBTN

Bouton cliquable de type tkinter.Button permettant de lancer le jeu

CGame : restartBTN

Bouton cliquable de type tkinter.Button permettant de recommencer le niveau

CGame : nextBTN

Bouton cliquable de type tkinter.Button permettant de passer au niveau suivant

CGame : pauseBTN

Bouton cliquable de type tkinter.Button permettant de mettre en pause la partie en cours

CGame : saveBTN

Bouton cliquable de type tkinter.Button permettant d'enregistrer la partie en cours

CGame : show_menuBTN

Bouton cliquable de type tkinter.Button permettant d'afficher le menu de parametre menu_setting_canvas

CGame : menu_popup_canvas

Cadre de type tkinter.Canvas correspondant au menu de la gestion des joueurs, il contient les différents boutons permettant de choisir/ajouter/supprimer un joueur (select_player_okBTN, add_playerBTN, remove_playerBTN)

CGame : player_lbFM

Cadre de type Frame contenant une tkinter.Listbox, utilisé pour dimensionner et placer correctement la tkinter.Listbox playerLB et la tkinter.Scrollbar player_lbSB

CGame : player_lbSB

Barre de défilement de type tkinter.Scrollbar permettant de naviguer dans la tkinter.Listbox playerLB

CGame : playerLB

Zone de liste de type tkinter.Listbox contenant tous les fichiers joueur chargés

CGame : select_player_okBTN

Bouton cliquable de type tkinter.Button permettant d'initialiser la jeu avec le joueur selectionné dans la zone de liste de type tkinter.Listbox playerLB

CGame : add_playerBTN

Bouton cliquable de type tkinter.Button permettant d'afficher un menu pour la création d'un nouveau joueur et ainsi ajouter le nouveau joueur dans la zone de liste playerLB

CGame : remove_playerBTN

Bouton cliquable de type tkinter.Button permettant la suppression d'un joueur

CGame : scoreLBL

Zone de texte de type tkinter.Label à l'interieur du cadre menu_popup_canvas qui affiche le score du joueur selectionné dans la zone de liste playerLB

CGame : player_niveauLBL

Zone de texte de type tkinter.Label à l'interieur du cadre menu_popup_canvas qui affiche le niveau du joueur selectionné dans la zone de liste playerLB

CGame : menu_setting_canvas

Cadre de type tkinter.Canvas correspondant au menu de paramètrage du jeu, il contient les différents boutons permettant de choisir la difficultée/l'enregistrement automatique ou pas/d'activer le son ou pas (menu_setting_difficulteeIMG, menu_setting_son_cb_var, menu_setting_enregistrement_value_RB) et d'appliquer les paramètres choisis (menu_setting_okBTN)

CGame : menu_setting_difficulteeLBL

Zone de texte de type tkinter.Label à l'interieur du cadre menu_setting_canvas qui affiche "Difficultée = "

CGame : img[]

Représente un tableau de ImageTk.PhotoImage qui contient 3 images affichées dans des tkinter.Label, elles correspondent aux images difficultées (vert : facile, orange : moyen, rouge : difficile)

CGame : menu_setting_difficulteeIMG[2]

Représente un tableau de tkinter.Label de taille 3, il permet l'affichage des images contenues dans le tableau img

CGame : menu_setting_son_cb_var

Variable de contrôle qui contient la valeur de menu_setting_sonCB (1:on, 0:off)

CGame : menu_setting_enregistrementLBL

Zone de texte de type tkinter.Label à l'interieur du cadre menu_setting_canvas, il affiche "Enregistrement = "

CGame : menu_setting_enregistrement_value_RB

Variable de contrôle qui contient la valeur sur l'état d'enregistrement (auto, manuel)

CGame : menu_setting_enregistrement_manuelRB

Bouton radio de type tkinter.Radiobutton permettant de changer la valeur de menu_setting_enregistrement_value_RB en 'manuel'

CGame : menu_setting_enregistrement_autoRB

Bouton radio de type tkinter.Radiobutton permettant de changer la valeur de menu_setting_enregistrement_value_RB en 'auto'

CGame : menu_setting_okBTN

Bouton cliquable de type tkinter.Button permettant d'appliquer les paramètres choisis par l'utilisateur et de fermer menu_setting_canvas

CGame : menu_jeu_info_canvas

Cadre de type tkinter.Canvas correspondant à l'état de la partie en cours, il contient le nom du joueur et le score (player_nameLBL, player_scoreLBL)

CGame : player_nameLBL

Zone de texte de type tkinter.Label à l'interieur du cadre menu_jeu_info_canvas qui affiche le nom du joueur actuel

CGame : player_scoreLBL

Zone de texte de type tkinter.Label à l'interieur du cadre menu_jeu_info_canvas qui affiche le score du joueur actuel

CGame : setting

Objet de type CSetting permettant la gestion, lecture, ecriture des différents fichiers (joueur et niveau)

CGame : playerIndex

Variable correspondant à l'index du joueur choisi, valeur par défaut -1

CGame : joueurTab[string, string, int, string, int]

Tableau contenant les informations de tous les joueurs avec comme format : [chemin_du_fichier, nom, score, partie_sauvegardée, niveau]

CGame : fichier_joueurTab[]

Tableau contenant le nom des fichiers joueur, il permet de savoir si un joueur a déjà été lu

CGame : game_running

Variable correspondant à l'état du jeu : True le jeu tourne, False le jeu n'est pas en marche. elle permet entre autre de lancer/stoper le jeu

CGame : objTab[CMap, CArrivee, CObstacle, [string, int, string, int]]

Tableau contenant les objets de la partie en cours avec pour format [CMap, CArrivee, CObstacle, [nom_joueur,score,sauvegarde,niveau_en_cours]]

CGame : engineTH

Thread qui gère le coeur du jeu quand la partie est lancée, il permet la gestion des collision avec les obstacles et si le joueur à gagnié ou perdu. Il s'arrête si game_running = False ou si le joueur à gagnié

CGame : sauvegarde_autoTH

Thread qui gère le traitement de la sauvegarde automatique (toutes les 1 minutes)

CGame : difficultee

Variable contenant la difficultée, facile=10, moyen=7, difficile=2, valeur_par_défaut=10
